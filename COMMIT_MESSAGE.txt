Fix loong64 Docker build - QEMU Java issue discovered, new workflow created

## 🔴 Critical Discovery

After extensive testing, discovered that **Java cannot run in QEMU-emulated loong64 environment**:

```
SR_initialize failed: Invalid argument
Error: Could not create the Java Virtual Machine.
```

This means:
- ❌ GitHub Actions + QEMU cannot produce usable loong64 Java images
- ❌ Even if Docker build "succeeds", containers cannot start
- ❌ All QEMU-based approaches are not viable

## ✅ Solution: New Workflow

Created **`build-jar-only.yml`** workflow that:
1. ✅ Compiles JAR on GitHub Actions (x86_64, fast & reliable)
2. ✅ Packages everything needed into a "build kit"
3. ✅ Includes Dockerfiles, scripts, and documentation
4. ✅ User downloads kit and builds on real loong64 hardware

This is the **only viable approach** for loong64 Java containers.
